# Strategy Report — 骨子草稿 (v2, 2026-06-21)

> 提出: Kaggle Writeup **≤2000語** ＋ Media Gallery（図表）。締切 2026-09-13。
> 評価: **Model 70%**（明快さ・独創性・**安定性**・**頑健性(運/初期非依存)**・LB成績）/ **Deck 20%**（コンセプト＋キーカード）/ **Report 10%**（構成・図表）。
> ⚠️ LB 上位は必須でない＝**独創的分析＋安定性**で勝負できる。我々の強みはここ。

## 0. タイトル/サブタイトル（案）
- 「**Reading the Meta, Not Just the Board**: 部分観測 TCG における相手モデリングと、**動的メタ三すくみのデータ駆動追従**」
- サブ: ルールベース・探索・模倣学習・RL を横断し、"なぜ素朴な探索/学習が効かないか"を実証し、**回転するメタにリプレイ解析で追従**して攻略する。

## 1. 要約 (Abstract) — ~150語
- 主張1: このゲームの本質は**部分観測下の相手モデリング**。placeholder 探索は有害、belief 接地で有益化（対照実験 0.083→0.417, 5倍）。
- 主張2: ラダーは**動的に回転する実ポケカのメタ三すくみ**（ex ビート ⇄ anti-ex 壁コントロール(Crustle) ⇄ 非exアタッカー）。
  リプレイ解析パイプラインで回転を**実測**し、各局面のカウンターを連続提出して追従（v003→v004→v006→v007）。
  現在は上位が**非ex アタッカーに収束**しつつあり、頂点デッキを複製＋専用方策でミラー強化（LB 1100超）。
- 主張3: 安定性(クラッシュ安全)＋高速手番がラダーレートを左右する。**学習と探索の両系統を実証で潰した**——
  実トップランカー319試合での **value 較正が中盤を当てられず（AUC<0.70, exp014）**、かつ **正確な near-terminal
  探索すらヒューリスティックを超えない（exp015）**。よって中位資源では「メタを読む rule-based＋良い操縦」が
  achievable ceiling であり最良 ROI。

## 3. 評価基盤と方法論 — ~200語
- ローカル対戦ハーネス（先後入替, 例外=反則負け, ~10ms/game）。固定相手プール平均勝率を指標化。
- **リプレイ解析パイプライン**（episodes/replay DL → 相手デッキ・勝敗・敗因抽出）＝メタ観測の生命線。
- 図: マッチアップ行列（公式4種＋random）, 強さ序列バー。

## 4. 中心的発見: 相手モデリングが探索の価値を決める ★独創性の核 — ~350語
- exp003/004/006: 素朴な先読み(0.21-0.27)/cold-start AlphaZero(0.03)/BC+MCTS(0.233) は**いずれもルールベース(0.68)未満**。
- 原因: **determinization が相手を placeholder(Snorlax)で穴埋め → 探索が"偽の相手"に最適化＝有害**。
- 解決: **belief 接地 determinization**（相手を実デッキからサンプリング）→ 対照実験で **belief 0.417 vs placeholder 0.083（5倍）**、相手の手数 50 vs 33。Dragapult に 0.667 で勝ち越し（素ルールベース超え）。
- 図: belief vs placeholder バー / 探索内の相手挙動比較 / PIMC マッチアップ。
- 補足: 公開上位notebookは探索 API を誤った署名で呼び実質無効＝**誰も探索を正しく使えていない**（差別化）。

## 5. 動的メタの追従と攻略: 回転する三すくみ ★独創性＋Deck Score の核 — ~420語
- **メタは静的でなく回転する**。リプレイ解析で日次の回転を実測:
  - 06-18: 上位 = **Crustle anti-ex 壁コントロール**（Dwebble→Crustle, 相手ex 完全無効, 120削り＋回復＋ドロー耐久）。
  - 06-20: 壁が廃れ **Lucario-ex ビートが復権**（自リプレイ field の 57%）。
  - 06-20 後半: 上位が **非ex アタッカーに収束**（#4 charmq・#8 tk が同型, ex を 0.69-0.92 で食い Crustle 遭遇0）。
- **三すくみの機序（実ポケカ対応）**: ex ビート → 勝つ → 壁(Safeguard control) → 勝つ → ex ビート ／ そして
  **非ex アタッカーが両方に勝つ＝頂点**: ①**単サイド**で ex(2-3サイド)にサイドレース勝ち ②非exダメージが
  Crustle の **ex限定 Safeguard を貫通**。壁が上位から押し出される動態まで観測。
- **データ駆動の追従（カウンターの連続提出）**:
  - v003 anti-Crustle（ex/megaEx が ex免疫に0ダメ→非ex Hariyama へ自動切替）: Crustle 0.10→0.55, **LB 1123**。
  - v004 Crustle 壁（メタ回転で ex 過半に対し）: 実戦で ex 0.62。
  - **v006 非ex apex**: トップ #4 の60枚を**リプレイから正確に複製**（`/extract-deck`）。ex 0.70/Crustle 0.80/全体0.68, **LB 1109**（歴代最高）。

- **デッキ解説（Deck Score）**: 非ex「Hop's Trevenant」アタッカー。キーカード＝**Hop's Snorlax「Extra Helpings」(全Hop's攻撃+30, ベンチ常在)**、

  すべて**単サイド**でサイドレースに強く、Safeguard を貫通する設計。

## 5b. 操縦の発見: 汎用方策と専用パッチ ★Model 独創性 — ~150語
- **汎用ルール方策が異種デッキを操縦できる**: lucario_v2 方策で Crustle(v004)・非ex(v006) を 0エラー運用。
- だが**精密局面（ミラー）は専用知識が要る**: 汎用は非exで `_base_attack` 不在→**攻撃 plan を作れず**
  Boss's Orders/リトリート/対象選択/KO認識が**全停止**していた（同一デッキでも操縦差で LB が変わる）。
- **v003 流の的を絞ったパッチ**で攻撃モデル(Extra Helpings/Choice Band/弱点)を注入→下流が一斉作動し、
  ミラー 0.60→**0.775**。"rewrite せず1モジュール注入"が安全（専用 full-rewrite の v005 は汎用に負けた前例）。
- 図: plan 形成前後の Boss's Orders 使用率 / ミラー smart-vs-generic。

## 7. 学習系への挑戦と現実 ★再現性・誠実さ — ~280語
- BC: 多教師でも greedy は弱い(0.22-0.39)＝誤差累積。ただし **warm-start 素材**。
- **RL Phase 2**（メタ相手プールで beat-the-field を狙う, 8世代）: 正直なネガティブ。loss↓だが
  eval 横ばい→**方策崩壊**（gen7 対 Lucario 0.0）。診断: ① 課題が sparse（all-ex で Crustle を倒すには
  非ex Hariyama への切替が必要だが探索で発見できず）、② checkpoint gating 不在で崩壊世代を採用。
- **RL Phase 3**（再設計: Lucario **ミラー**特化。理由＝ミラーは勝てる・field の 57%・**belief が完全**で
  探索の最良ケース・v003 の弱点を直撃。安定化＝固定相手で value 接地＋経験リプレイ＋崩壊復帰 gating）:
  gating で崩壊は防げたが、ミラー勝率は **0.31 止まり**（目標 0.55, v003 ref 0.47 に未達）。
- **最重要の内訳**: BC greedy 0.17 → BC+belief-MCTS(search24) **0.31**。**伸ばしたのは「学習」でなく
  「推論時の探索」**（exp008 の再確認）。RL 微調整自体は eval を改善しなかった。
- **探索量スケーリング（決定的診断）**: ミラー勝率は search24=0.31 → 48=0.19 → 96=0.13 と
  **探索を増やすほど悪化**（速度は 0.05-0.18s/手で無問題）。良い value 関数なら探索は単調に効くので、
  これは **学習した value 推定が不良**で、深い探索が誤推定を増幅している証拠。ボトルネックは
  速度でも探索量でもなく **value ネットの品質**。
- 結論（3本の証拠で検証された誠実な負）: ①Phase2 崩壊 ②Phase3 ミラー 0.31 天井 ③探索増で悪化。
  **中位資源では warm-start belief-MCTS は強い rule-based を超えない**。価値の源泉は探索＋正確な
  相手モデル(belief)で、学習の寄与は限定的。残された原理的仮説は「value を較正 → MCTS に載せる」。
- **exp014: その unblocker を最良データで検証 → 決定的な負（4本目）★**。
  上記の「value 較正すれば探索が効く」仮説を、**ヒューリスティック自己対戦でなく実トップランカーの
  319試合（実勝敗ラベル）**で検証。試合単位 holdout（leakage なし）で value ネット(obs→P(勝))を学習し、
  **探索が最も効くべき中盤(進行 0.4-0.6)を当てられるか**を go/no-go に設定。
  - **2つの独立な特徴量設計が一致して不合格**: ①strategy-lens スカラー17個 = 中盤 AUC **0.637**（＝
    prize差単独 0.688 と同値＝盤面スカラーは prize 数を超える情報なし）、②＋**手札中身＋自他盤面の
    カードレベル embedding** = 中盤 AUC **0.585**（train AUC 0.999＝特定試合の丸暗記で汎化せず）。
  - 終盤(進行0.75+)は AUC **0.80** で当たるが、**中盤は near-random**。
  - **これが exp010「探索増で悪化」を機構的に説明**: value は中盤で無情報 → MCTS がノイズを増幅。
    PTCG 中盤の高い結果分散はゲーム固有（運・ドロー）であり、モデル不足ではない。
  - **帰結**: deep-RL/MCTS は「未試行」でなく**経験的に上限**。価値の源泉が「学習」でなく「推論時探索＋
    相手モデル」であること（exp008/exp010）と整合し、**なぜ中位資源では『メタを読む rule-based＋良い操縦』が
    最良 ROI か**を実証で締める。
- **exp015: 正確な near-terminal 探索すら超えない（探索系統も上限）★**。
  exp014 の「終盤は AUC 0.80 で読める」を受け、学習せず**エンジンの正確な前方探索を自ターン(near-terminal)
  限定**で使い、v008 が取りこぼす KO/リーサルを拾う戦術レイヤーを実装（自ターンは相手が動かず手札も見える＝
  ほぼ完全情報で、exp008 の placeholder 問題が無関係になる点が鍵）。
  - 3変種（プライズ最大化 / リーサル限定 / K=5 頑健リーサル）すべてミラーで **≤0.47**＝**v008 を超えない**。
  - 原因: ①自ターンも自分のドローで真には非決定的（偽陽性リーサル）②貪欲な near-terminal 最適化が
    ヒューリスティックの一貫した多ターン狙撃プランを壊す ③v008 は既にリーサルを十分拾っている。
  - **学習(exp014)と探索(exp015)の両系統を実証で潰した** → 中位資源では rule-based＋良い操縦が achievable ceiling。
- 図: BC スケール曲線 / Phase 2 崩壊（loss↓ eval→0）vs Phase 3 gating（崩壊抑止）/ **探索量スケーリング（増で悪化）** /
  **exp014 value 較正の進行段階別 AUC（中盤<0.70・終盤0.80）＋スカラー vs カードレベルの一致** / **exp015 戦術探索 3変種ミラー≤0.47**。

## 8. 結論と今後 — ~150語
- 鍵は「盤面だけでなく**回転するメタ**を読む」＝相手モデリング＋動的メタ追従。
  リプレイ解析で回転を実測し、頂点デッキを複製＋専用方策で操縦して、中位資源でも上位に迫る。
- 今後: 非exミラーの更なる操縦改善, dragapult/スプレッド耐性, アーキタイプ推定の自動化,
  デッキ革新（メタ回転に刺さる新アーキを模倣でなく自作）。
  **RL は「value 較正の不成立」を実データで確認済み**——再挑戦するなら単一状態の value 回帰でなく、
  系列モデル/不確実性込みの評価か、根本的に異なる枠組みが要る（中位資源では非推奨）。
