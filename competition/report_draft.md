# Strategy Report — 骨子草稿 (v0, 2026-06-19)

> 提出: Kaggle Writeup **≤2000語** ＋ Media Gallery（図表）。締切 2026-09-13。
> 評価: **Model 70%**（明快さ・独創性・**安定性**・**頑健性(運/初期非依存)**・LB成績）/ **Deck 20%**（コンセプト＋キーカード）/ **Report 10%**（構成・図表）。
> ⚠️ LB 上位は必須でない＝**独創的分析＋安定性**で勝負できる。我々の強みはここ。

## 0. タイトル/サブタイトル（案）
- 「**Reading the Meta, Not Just the Board**: 部分観測 TCG における相手モデリングと、メタ三すくみのデータ駆動攻略」
- サブ: ルールベース・探索・模倣学習・RL を横断し、"なぜ素朴な探索/学習が効かないか"を実証して攻略する。

## 1. 要約 (Abstract) — ~150語
- 主張1: このゲームの本質は**部分観測下の相手モデリング**。placeholder 探索は有害、belief 接地で有益化（対照実験 0.083→0.417, 5倍）。
- 主張2: ラダーは**実ポケカのメタ三すくみ**（ex ビート → anti-ex 壁コントロール(Crustle) → 非exアタッカー）。リプレイ解析で特定し、1点修正のカウンターで **LB 1123**（公開最高909超）。
- 主張3: 安定性(クラッシュ安全)がラダーレートを左右する（公開勢の +64 はほぼ安定性）。

## 2. 問題設定 (Game & Challenge) — ~200語
- cabt エンジン / 部分観測・確率的・可変異種の行動空間（indices を返す契約 → 固定ヘッド RL が不向き）。
- TrueSkill ラダー（μ0=600, 勝敗のみ, 最新2提出, ライブ継続対戦）。10分/試合。
- 図: 観測/行動の構造、行動空間ヒストグラム。

## 3. 評価基盤と方法論 — ~200語
- ローカル対戦ハーネス（先後入替, 例外=反則負け, ~10ms/game）。固定相手プール平均勝率を指標化。
- **リプレイ解析パイプライン**（episodes/replay DL → 相手デッキ・勝敗・敗因抽出）＝メタ観測の生命線。
- 図: マッチアップ行列（公式4種＋random）, 強さ序列バー。

## 4. 中心的発見: 相手モデリングが探索の価値を決める ★独創性の核 — ~350語
- exp003/004/006: 素朴な先読み(0.21-0.27)/cold-start AlphaZero(0.03)/BC+MCTS(0.233) は**いずれもルールベース(0.68)未満**。
- 原因: **determinization が相手を placeholder(Snorlax)で穴埋め → 探索が"偽の相手"に最適化＝有害**。
- 解決: **belief 接地 determinization**（相手を実デッキからサンプリング）→ 対照実験で **belief 0.417 vs placeholder 0.083（5倍）**、相手の手数 50 vs 33。Dragapult に 0.667 で勝ち越し（素ルールベース超え）。
- 図: belief vs placeholder バー / 探索内の相手挙動比較 / PIMC マッチアップ。
- 補足: 公開上位notebookは探索 API を誤った署名で呼び実質無効＝**誰も探索を正しく使えていない**（差別化）。

## 5. メタゲーム分析と攻略: 実ポケカの三すくみ ★Deck Score の核 — ~350語
- リーダーボード＋自リプレイで **トップ＝Crustle anti-ex 壁コントロール**（Dwebble→Crustle, 相手ex 完全無効, 120削り＋回復＋ドローで耐久）を特定。
- **実ポケカ対応**: Mimikyu/Sylveon 系「Safeguard 壁コントロール」。テックに非ex炎アタッカーを足す進化＝現実の「コントロールに勝ち筋を追加」。
- **三すくみ**: ex ビート(Lucario) → anti-ex 壁(Crustle) → 非exアタッカー(我々の v003)。
- **カウンター(v003)**: 「ex/megaEx は ex-免疫に0ダメ」を方策に教え、自動で非ex Hariyama(210, Crustle一撃)へ。Crustle 戦 0.10→0.55、**LB 1123**。
- **模倣(v004/v005)**: Crustle control の再現。汎用方策では操縦不全(627)、専用方策で改善(835)。
- 図: 三すくみ図 / prize liability 比較（Mega Lucario ex=3渡しで脆い） / v003 のマッチアップ（Lucario 6-0, Crustle 5-4）。

## 6. 安定性と運用 ★Model の安定性軸 — ~200語
- クラッシュ安全ラッパー（例外/不正手→合法フォールバック）＝検証(ミラー)通過・反則負け回避。公開勢の +64 はほぼこれ。
- 高速(0.02-0.16s/手)でタイムアウト無縁（探索版 PIMC は ~30s で重く非実用）。
- 運用サイクル: 週次メタ監視→カウンター更新, 裏で RL 継続学習, 随時レポート更新。
- 図: 安定性(0エラー)とLBの関係 / レート収束曲線。

## 7. 学習系への挑戦と現実 ★再現性・誠実さ — ~280語
- BC: 多教師でも greedy は弱い(0.22-0.39)＝誤差累積。ただし **warm-start 素材**。
- **RL Phase 2**（メタ相手プールで beat-the-field を狙う, 8世代）: 正直なネガティブ。loss↓だが
  eval 横ばい→**方策崩壊**（gen7 対 Lucario 0.0）。診断: ① 課題が sparse（all-ex で Crustle を倒すには
  非ex Hariyama への切替が必要だが探索で発見できず）、② checkpoint gating 不在で崩壊世代を採用。
- **RL Phase 3**（再設計: Lucario **ミラー**特化。理由＝ミラーは勝てる・field の 57%・**belief が完全**で
  探索の最良ケース・v003 の弱点を直撃。安定化＝固定相手で value 接地＋経験リプレイ＋崩壊復帰 gating）:
  gating で崩壊は防げたが、ミラー勝率は **0.31 止まり**（目標 0.55, v003 ref 0.47 に未達）。
- **最重要の内訳**: BC greedy 0.17 → BC+belief-MCTS(search24) **0.31**。**伸ばしたのは「学習」でなく
  「推論時の探索」**（exp008 の再確認）。RL 微調整自体は eval を改善しなかった。
- **探索量スケーリング（決定的診断）**: ミラー勝率は search24=0.31 → 48=0.19 → 96=0.13 と
  **探索を増やすほど悪化**（速度は 0.05-0.18s/手で無問題）。良い value 関数なら探索は単調に効くので、
  これは **学習した value 推定が不良**で、深い探索が誤推定を増幅している証拠。ボトルネックは
  速度でも探索量でもなく **value ネットの品質**。
- 結論（3本の証拠で検証された誠実な負）: ①Phase2 崩壊 ②Phase3 ミラー 0.31 天井 ③探索増で悪化。
  **中位資源では warm-start belief-MCTS は強い rule-based を超えない**。価値の源泉は探索＋正確な
  相手モデル(belief)で、学習の寄与は限定的。原理的な次手は「stock vs stock の大量対戦で value を
  較正 → MCTS に載せる」だが、より大きな再設計＝後半の計算資源投入時の課題。
- 図: BC スケール曲線 / Phase 2 崩壊（loss↓ eval→0）vs Phase 3 gating（崩壊抑止）/ **探索量スケーリング（増で悪化）**。

## 8. 結論と今後 — ~150語
- 鍵は「盤面だけでなくメタを読む」＝相手モデリング。安定性とメタ適応で中位資源でも上位に迫れる。
- 今後: belief-MCTS の高速化, アーキタイプ推定, 計算資源を活かした RL の本格化。
