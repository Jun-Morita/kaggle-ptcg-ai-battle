# exp006_bc — SESSION NOTES

## 仮説
強ルールベース lucario_v2 を教師に、その意思決定 (obs→選択) を模倣学習(BC)させたネットなら、
(a) 探索なしで高速に教師(プール0.680)へ迫り、(b) MCTS の warm-start に使えて exp004(cold-start 失敗)を救える。

## 実装（exp004 のモデル/特徴量を再利用）
- `train_bc.py`: lucario_v2 でゲームを回し各決定の (encoder特徴, 候補手列, decoder特徴, 教師が選んだindex, 勝敗) を記録。
  ミラー(両者記録)＋ vs プール(教師側のみ記録, 相手多様化)。policy=交差エントロピー, value=Huber で学習。
- `eval_bc.py`: 学習ネットを **探索なし argmax** でagent化（クラッシュ安全, 0.1s/game）→ exp002 プール評価。
- データ生成は高速（10ゲーム→933サンプル/1秒, 400ゲーム→37804サンプル/39秒）。

## 結果

### run2 (800ゲーム, 30エポック) — データ増で改善
- 模倣精度 0.848（run1 0.767 から向上）。**BC単体プール平均 0.389**（run1 0.278 から改善）。
  vs random 0.900 / dragapult 0.200 / lucario_v1 0.600 / lucario_v2(教師) 0.367。
- **BC はデータ量で単調にスケール**（acc 0.767→0.848 で pool 0.278→0.389、対教師 0.167→0.367）。
  教師(0.68)には未達だが伸長中。最弱マッチは vs dragapult(攻撃型)。

### run1 (400ゲーム, 15エポック)
- 模倣精度: epoch0 0.614 → epoch14 0.767（まだ上昇中）。value_loss 0.243→0.103。
- **BC単体（探索なし, n=30/相手）プール評価**:
  | 相手 | 勝率 |
  |---|---|
  | random | 0.867 |
  | dragapult | 0.267 |
  | lucario_v1 | 0.400 |
  | lucario_v2(教師) | 0.167 |
  | **対ルールベース平均** | **0.278** |
- **BC warm-start MCTS（search16, n=20）= 0.233**（BC単体より悪化, かつ15倍遅い1.5s/game）。

## 重要な発見
1. **BC単体(0.278) は cold-start MCTS(0.03)の約10倍強い**＝教師の知識がネットに入っている。学習路線として最良。
   ただし教師(0.68)には届かない＝**BC の誤差累積**（1手0.77精度でも30-150手で複利的に劣化、難所での誤りが致命的）。
2. **MCTS warm-start は逆効果（0.278→0.233）**。exp003/004 と同根の根本原因＝
   **部分観測下で determinization が相手を placeholder(Snorlax)で穴埋め → 探索が「偽の相手に最適な手」へ
   良いBC方策を歪める**。このゲームでは「相手モデルの質」が探索の価値を決め、現状の穴埋めでは search が有害。

## 結論 / 方針
- **学習エージェントは総じてルールベースに未達**。最強は依然 ルールベース lucario_v2(0.680)＝提出済 v001。
- 探索を活かすには **determinization の相手モデル改善が必須**（meta デッキ仮定 or 相手をルールベースで rollout）。これは独立した研究課題。
- 当面の競争力は **(Lever1) デッキ最適化** と **ルールベース改良** が最有望（探索問題を回避でき、公開notebookも最大効果と明言）。
- BC の価値: 高速・頑健な fast policy、かつ **Strategy レポートの独自性**（「部分観測下で素朴な探索/BC+探索がなぜ効かないか＝相手モデルがボトルネック」という知見は説得力がある）。
