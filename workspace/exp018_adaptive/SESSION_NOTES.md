# exp018_adaptive — SESSION NOTES

問い（ユーザー仮説）: 実ポケカのように「相手アーキタイプを見極め方策を切り替える」機構がないと勝てないのでは？
→ まずトップランカーが実際にそうしているかをデータで検証。

## 方法（`analyze_adaptation.py`）
トップ非ex選手のキャッシュ済み実リプレイを**相手アーキタイプ別**に分解し、各マッチアップで
W-L / avg_bench（展開度＝prize-liability）/ attack-per-turn（テンポ）/ **我々の方策との決定一致率**を集計。

## 結果（cached replays）
**Debauchery #1（65試合, 現トップ）:**
| 相手 | W-L(wr) | avg_bench | attack/turn | match_vs_us |
|---|---|---|---|---|
| 非ex(ミラー) | 18-10 (0.64) | 3.15 | 0.20 | 0.28 |
| lucario_ex | 18-5 (0.78) | 2.94 | 0.17 | 0.27 |
| crustle | 3-0 (1.00) | 3.53 | 0.21 | 0.29 |
| dragapult | 2-0 (1.00) | 2.94 | 0.33 | 0.30 |
| Alakazam(mixed_ex1) | 2-1 (0.67) | 1.77 | 0.00 | 0.28 |

charmq(25試合): ミラー 0-1, ex 0.62, crustle 0.83, bench 3.7-4.4。tk(23試合): ex 0.92 だが**ミラー 1-8(0.11)**＝上位内でも差。

## 結論（ユーザー仮説への答え＝一部反証）
1. **トップの強さは「相手別の方策切替」ではない**: 我々との決定一致率が**全マッチアップで一様に低い(~0.27-0.30)**。
   相手別切替で勝っているなら一致率はマッチアップ間で偏るはずだが、**どの相手でも等しく違う**＝彼らは相手別の特別手でなく
   **一貫した別スタイル**で打っている。相手別の挙動変化も穏やか(bench 2.9-3.5, attack 0.17-0.33)＝劇的なプラン切替でない。
2. **彼らの真の edge = 一貫した規律、特に prize-liability 管理**: ベンチを少なく展開(~3)／我々は多い(~4+)。
   ベンチ過剰＝倒される的増＝**サイド献上**。exp013 の「我々は Phantump 過剰展開」を**定量裏付け**。
3. **Debauchery は我々が落とす非exミラーを 0.64 で勝つ**。勝因は相手読みでなくこの規律＋資源温存。

## 含意 / 次
- **「相手アーキタイプ適応レイヤー」はトップの強さの源泉でない**（データ）→ 優先度を下げる。
- **本命 = prize-liability 規律**（ベンチ過剰展開抑制／空きスロット維持／Phantump 乱発抑制／資源温存）を方策に注入し、
  **実物スモーク（汚染しない信頼ハーネス, build_submission.py）**で **非exミラー(我々0.38) が上がるか**を検証。
  - 注意: ベンチ抑制しすぎると Phantump→Trevenant 進化数が減り打点低下のリスク → スモークで両にらみ。
- 留意: サンプル小（charmq25/tk23, dragapult は Debauchery 2試合のみ）。傾向の方向性として扱う。
- 再利用資産: `analyze_adaptation.py`（任意の上位×デッキ×相手別の挙動分解）。

## 規律パッチ実装＆検証（`discipline_policy.py`）→ ★v008 上位互換
非ex の `_score_play_pokemon` は base で全ポケ 20000＝無条件ベンチ（過剰展開の原因）。実プレイ指標を
トリガー化して上書き:
- **ベンチ/prize-liability**: Trevenant ライン上限 ~3、空きスロット維持、Phantump 乱発抑制、Snorlax/Cramorant は1枚、
  Dunsparce 系は ~1ライン。**サイド劣勢なら +1 攻撃役を許容**（指標=自他プライズ差）。
- **エネ過剰ガード**: 1エネ攻撃役(Trevenant/Phantump/Cramorant)に2個目を付けない。
- **相手指標（壁検出）**: 相手盤面に Crustle/Dwebble があれば規律を解除し base 展開（壁は割るのに展開が要る）。
  ※ただし Crustle は中盤出現で序盤判断に間に合わず効果限定的（下記 Crustle は元々退行でなかった）。

### 検証（built artifact, 独立実行＝汚染なし）
- **非exミラー（discipline vs v008, n=200）= 0.550（110-88-2）** ＝95%CI が 0.50 を除外＝**実在の改善**。
  v008 のミラー上限を初突破（exp013 v009=0.50, exp015 戦術=≤0.47 はいずれも未達）。
- **公平対比（n=80, 同一相手, 両 built）**: ex +0.00(0.762=0.762) / Crustle -0.01(0.713 vs 0.725, 誤差) / dragapult +0.05(0.125 vs 0.075)。
  ※ 以前の「Crustle 0.938→0.725 退行」は n=16 スモークの偶然＝**誤報**。v008 の実 Crustle は ~0.72。
- クラッシュ0・高速(0.36s/手 max)。
- **結論**: discipline = **v008 の正当な上位互換**（全マッチアップ改善 or 同等、ミラー +0.05）。

### 学び（ユーザー仮説の最終回答）
- 「相手アーキタイプで方策を切替」単体はトップの edge でない（前段）。だが**「一貫した規律（prize-liability）」＋
  少数の相手指標トリガー（壁検出）**は効く。**学習・探索で破れなかったミラー上限を、実プレイ指標のトリガー化で突破**。
- 残課題: 改善は modest(+0.05)。実ラダーのミラー 0.38（実相手は上手い）への転移は不確実。
- 提案: discipline を **v009 として提出候補**（eligible {v009 discipline, v008}, strictly ≥ v008）。ユーザー承認後。
