# exp033 — value-guided guard（Stage 2b）: v013 に及ばず、ただし構造は健全

## 設計
v013 の3値 doom 拒否権を、exp032 価値ネット（純numpy MLP, 中盤AUC 0.773）による
連続スコアに置換: MAIN で base＋最大8候補 × K=4 belief rollout → 終端 value 平均、
`best ≥ base + margin(0.10)` でのみ上書き。発火 ~7回/試合、0エラー、~10s/試合（本番予算内）。

## 解釈
- 価値誘導は基準よりは上（+0.055）だが **v013 の doom 拒否権に総合で及ばない**。
- **crustle 退行の機構仮説 = OOD**: 学習プール（v_trev/charmq/lucario/grimmsnarl）に
  crustle 系ウォールが無い → 壁相手の状態評価が分布外で誤誘導。
  （archaludon もプール外だが 0.190 と退行せず — 元々床のマッチで判別力低）
- ミラー +0.045 は価値誘導の見どころ（doom 拒否権はミラーで弱かった）。
- **判定: v013 維持。** 価値ネットは「学習可能」(exp032) だが、この統合では実戦利得に未変換。

## 次の一手（安価な OOD 修正）
datagen プールに crustle / archaludon / LO(GreatTusk) デッキを追加 → ~10万試合再生成（~1h）→
再学習 → crustle 退行が消えるかだけ n=200 で再テスト。消えて合計が v013 を超えたら v014。
超えなければ「doom 拒否権 = 実用最適」で確定し Stage 2 本体（選択的 PIMC の別軸）へ。

## OOD 修正イテレーション（2026-07-04）→ 不発、価値ライン終結
- プール 4→8 デッキ（crustle/archaludon/lo_tusk/dragapult 追加）で 10万試合再生成（data2/, 2.54M行）、
  新旧結合 ~5M行で再学習（game ID 衝突はディレクトリ接頭辞で回避）→ **中盤 AUC 0.806**（0.773 から改善）。
- crustle ピンポイント再テスト n=200: **0.715**（修正前 0.700, 基準 0.765, v013 0.795）→ **直らず**。
- **切り分け完了**: 原因は OOD でなく統合方法。「1応手先の価値最大化」は壁マッチの多ターン計画に対して
  近視眼的（exp015 と同型）。margin 0.10 でも有害発火を防げない。
- **結論**: 価値は学習可能(0.806) だが horizon-1 value override では利得化できない。
  **v013 doom 拒否権 = 実用最適で確定**。価値ネット資産（value_mlp2.npz, datagen, native engine）は保持
  — 将来の使い所は「多ターン計画を壊さない統合」（例: 候補が同 doom 級の時のタイブレークのみ、
  または PIMC 終端の補助評価）だが、期待値は限定的と評価。
