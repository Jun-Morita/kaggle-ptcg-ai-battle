# exp036 — GA によるパイロット定数進化 → 過学習で打ち切り

## 設計
lucario_v2+gust+revenge チェーンの中核12定数を log2 倍率ゲノム化（恒等性テスト合格: genome=0
が v012 と547決定で完全一致）。集団24/エリート4/トーナメント選択、ラダーシェア加重プール
n=48/個体、40世代、ホールドアウト監視付き。

## 結果
- Champion fitness(訓練時) 0.734、ホールドアウト良好(0.75) → **しかし n=200 厳密検証で瓦解**。
- 実測合計 2.48（v012同水準2.45、**v014の2.67に遠く届かず**）。paired vs v012 = 0.490（互角）。
- 全マッチ v014 以下、退行はしていないが上積みもゼロ。0 crash のみ確認。

## 診断
- fitness の n=48/個体（各マッチ4-14試合, SE 0.15-0.25）は選択圧の分解能に対して不足。
  40世代×24個体の選択は**運の良い乱数シードに適応**していた（exp010/014 の過学習パターンの再来）。
- **構造的な限界**: GA が動かせるのは静的スコア定数のみ。v014(turn-beam) の優位は
  「ターン系列を engine で正確に検証してから採用する」という**探索の質**そのものにあり、
  静的定数の再チューニングでは原理的に届かない範囲。

## 判断
**打ち切り。v014 を正として維持**。GA というアプローチ自体は（恒等性テスト・チェックポイント
再開・並列評価基盤含め）技術的には健全に機能したが、この壁は「もっと大きい n で再挑戦」より
「探索を強化する」方向に資源を割く方が期待値が高いと判断。

## 資産（将来の再利用に足るもの）
- `genome_policy.py` の倍率ゲノム機構＋恒等性テストパターンは、将来 turn-beam 内の評価関数
  重みを調整する用途（EVOLVE→スコア関数のみ）に転用可能。
- `evolve.py` のチェックポイント/自動再開/ホールドアウト監視パターンは長時間実験の標準形として再利用可。
