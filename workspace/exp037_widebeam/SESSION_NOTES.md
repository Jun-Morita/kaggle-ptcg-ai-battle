# exp037 — turn-beam の探索予算拡張（TB_K/BEAM/BRANCH/MAXSTEPS スイープ）

## 動機
v014 は持ち時間 600s のうち <1% しか使っていない。同じ「検証付き上書き」設計のまま、
beam/branch/depth を数倍に広げるだけで質が上がるかを検証。

## 設定
TB_K=3(2→3), TB_BEAM=10(5→10), TB_BRANCH=16(10→16), TB_MAXSTEPS=3000(900→3000)。
タイミング 1-3.6s/試合（予算内）。

## 結果（n=200, 0 crash）
| matchup | v014(既定) | wide config |
|---|---|---|
| ex_lucario | 0.770 | **0.800** |
| dragapult | 0.220 | 0.215 |
| archaludon | 0.195 | 0.200 |
| mirror_chq | 0.580 | 0.580 |
| crustle | 0.905 | 0.855 |
| 合計 | 2.670 | 2.650 |

## 結論: 予算拡張だけでは頭打ち
誤差内の同水準。**「探索の量」より「探索の質（設計）」が効いていた**ことの追認 —
turn-beam の勝因は beam/branch のサイズでなく「検証付き上書き」という設計原理そのもの。
次の伸びしろは exp038（相手モデル修正・評価関数拡張・αβ法・move ordering）に委ねる。
