# exp045 — turn-beam終端の学習評価器: 手書き(pz, dmg)タプルの置換

## 背景（RL設計再考、2026-07-11ユーザー合意）
13件のRL負の結果の3軸棚卸し（行動・状態・報酬）で合意した本命プラン:
- v014の強さの源泉はturn-beamの**手書き1ターン報酬 (取得サイド, 与ダメージ+30)**。
- exp044でこのタプルの盲点を実データで特定済み: **ダメージ・タイブレークは320HP本体への
  削りを好み、進化ライン狩り（Dreepy/Drakloakを狩って将来のDragapult ex数を減らす）の
  将来価値が一切見えない**。Boss's Ordersの浪費（KOできない大物を引いて削る）も同根。
- 提案: ビーム終端状態の評価を**学習した価値関数**に差し替える。学習の単位を
  「ターン計画の選択」に上げる（行動集合の再考）＋ 終端±1より密な信号（報酬の再考）。

## exp033との構造的差別化（最重要の設計論点）
exp033（value-guided guard）は**負けた**（2.505 vs v013の2.58、crustleで悪化）。
その敗因は「horizon-1の貪欲上書き＝多ターンの積み上げ計画を単発の価値最大化が壊す」。
本実験は同じ価値関数でも統合構造が異なる:
- v014の**検証付き上書き規律は不変**（ビームが列挙した完全なターン系列同士の比較のみ、
  全K決定化で確実に上回る時のみ発火）。
- 置換するのは**タイブレークの部分だけ**: 第一キー pz（このターンの取得サイド）は保持し、
  第二キーの「与ダメージ」を「終端状態の学習価値」に置き換える。
  → 「サイドが取れるなら取る」は変わらず、「サイドが取れないターンに何を仕込むか」
  の質だけが変わる（exp044の診断した盲点はまさにここ）。

## v1設計: 学習ゼロで検証（exp033資産の再利用）
- 価値関数: `exp032_valuescale/value_mlp2.npz`（17特徴→MLP、中盤AUC 0.806、
  8デッキ混合・約20万試合・500万行で学習済み）をそのまま使用。
- 特徴抽出: `exp033_valueguard/value_guard.py`の`_feats()`（17スカラー、typed obs対応）を
  流用。ビーム終端のsearch_state観測にそのまま適用可能。
- 統合: `tb_patch.py`の`outcome(ss)`を env-gate `TB_VALUE=1` で
  `(pz, dmg)` → `(pz, V(終端状態))` に差し替え。勝利=+100 pzの既存ロジックは不変。
  比較の`_TB_DMG_MIN`マージンは価値スケール用に別マージン（`TB_VALUE_MARGIN`）へ。
- 推論コスト: 17特徴のMLPはマイクロ秒級。ビーム内で50-100回/決定呼んでも無視できる
  （純Python化も trivial、出荷時はexport_pureパターン）。

### v1実装・field評価結果 → NO-GO（exp033と同型の再発、2026-07-11）
`tb_patch.py`に`TB_VALUE=1`ゲートを実装。`outcome(ss)`の第二キー(dmg)を
`exp032_valuescale/value_mlp2.npz`ベースの終端状態価値(×1000)に置換、第一キー(pz、
プライズ差)は不変。マージンは`TB_VALUE_MARGIN`(既定20)に分離。OFF時はコンパイル確認
でTB_VALUE=0の死コード分岐のみ、numpyインポートも実行されないことを確認。

スモークn=4×5エラー0を確認後、field n=200×5実行:

| マッチアップ | TB_VALUE | v014基準 | Δ |
|---|---:|---:|---:|
| crustle | 0.905 | 0.905 | ±0.00 |
| ex_lucario | 0.795 | 0.770 | +0.025 |
| dragapult | 0.185 | 0.170 | +0.015 |
| **archaludon** | **0.135** | **0.195** | **-0.06** |
| mirror_chq | 0.600 | 0.585 | +0.015 |
| **合計** | **2.620** | **2.670** | **-0.05** |

**判定: NO-GO（合計で悪化、pairedに進む前にfieldで足切り）。**
archaludonの悪化(-0.06)が他の小幅な改善(ex_lucario/dragapult/mirror、いずれも
+0.015〜+0.025)を打ち消した。crustleは無傷(±0.00)——**exp033で見た「壁/アンチex
マッチアップで価値関数が悪化させる」パターンが、統合構造(タイブレークのみ置換・
上書き規律は保持)を変えても再発**。exp033はcrustle(0.905)で崩れたが今回はcrustleは
無傷でarchaludonが崩れた、という違いはあるものの、根本原因は同じ可能性が高い:
**value_mlp2(17特徴、8デッキ混合合成データ学習)自体がこの種のマッチアップの判断に
不向き**（アンチex/壁系の勝ち筋は「短期的にHPやプライズが不利に見えても持久戦に
勝てる」という価値関数の17特徴では捉えにくい構造を持つ可能性）。

**次の一手（未着手）**: (a) archaludonを含む多様なデッキでvalue_mlp2を再学習
（exp033のOOD対策と同型、当時は効果薄だった）、(b) この価値関数を捨て、
turn-beam専用に(pz,dmg)に代わる**手書きの改良版タイブレーク**（進化ライン狩り等
exp044の知見を定数として反映）を試す、(c) このプラン2自体を畳み、プラン3
（encoder語追加によるBCの状態表現拡張）に集中する。誠実にNO-GOとして記録し、
paired評価は実施しない（fieldで既に足切り基準を満たさないため）。
