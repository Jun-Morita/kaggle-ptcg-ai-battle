## exp048(a) 本番学習 → 決定的ゲートで惨敗、誠実な負の結果（2026-07-13, GPU解放後）

GPU解放を確認（RTX 3060, 3%使用率）後、即座に本番学習を実行:
```
uv run python pretrain.py --epochs 8 --d-model 128 --glob data/starmie_w1.pkl --tag starmie_pre1 --opp-drop 0.3
```
8エポック、4-7秒/エポック（GPU、30,152決定のみなので高速）。学習曲線は健全に単調改善:
acc.all(oracle_free) 0.535→0.602→0.614→...→**epoch6で0.645（ピーク）**→epoch7 0.635
（軽微な過学習、epoch6を採用）。mixed_ex4(自デッキの実アーキタイプ)0.677、
non_ex_attackers 0.652、crustle_control 0.564。全チェックポイント保存済み。

**決定的ゲート（`eval_bc_starmie.py`, v016壁=Crustle系, exp048(b)と同一相手・同一デッキ）**:
- oracle_free=True（出荷相当条件）: n=20, **winrate=0.050（1勝19敗）**
- oracle_free=False（相手デッキ既知、参考条件）: n=20, **winrate=0.100（2勝18敗）**

**両条件とも既存の2基準を大幅に下回る**:
| pilot | vs v016壁 winrate |
|---|---:|
| 汎用lucario_v2パイロット (exp048(b)以前) | 0.825 |
| ハンドパッチ済みヒューリスティック (exp048(b)) | 0.933 |
| **BC(exp048(a), 本結果)** | **0.05-0.10** |

トレース確認（1ゲーム, 43手）: 選択インデックスは0固定ではなく候補間で多様に変化
（5,7,2,8,9等）——退化した定数方策のバグではなく、**ゲーム全体を通した一貫性のある
競技レベルの操縦ができていない**という能力不足の結果と判断。

**診断・教訓**: 単一決定精度0.645（悪くない）にもかかわらずゲーム全体では惨敗——
本セッションで繰り返し確認してきた「decision-match ≠ strength」則の最新の再確認
（exp042/048(b)/050/PokéForgeポストモーテムに続く）。決定的な違いは**データ規模**:
本線RLネット(exp041)はまず5.9M件の合成自己対戦データで事前学習してから実ラダー
データで微調整する経路を辿ったのに対し、本実験は**わずか30,152決定（807試合、
実質2人のプレイヤー）からゼロから学習**——精度0.645止まりで、ゲーム全体を通した
一貫性のある操縦には全く届かなかった。裏付け: 本線ネットの合成事前学習後の
oracle_free精度は0.83（本結果の0.645を大きく上回る）。

**判定: NO-GO。** exp048(a)のBCラインはこのデータ規模では竜頭蛇尾に終わった。
再挑戦する場合の選択肢（未実施）: (1) 本線の5.9M合成事前学習チェックポイントを
初期値にして、この30K件の実データで微調整（ゼロから学習でなく転移学習にする）、
(2) より多くの実データを集める（tomatomato/taksaiの過去キャッシュをさらに拡張）、
(3) BCを諦め、既存のハンドパッチ済みヒューリスティック(0.933)を実運用の到達点として
確定する。ヒューリスティックの0.933は既に汎用パイロットを大きく上回っており、
現時点でこのラインへの投資対効果は低いと判断。次のGPU投資は本線RL
（v019/v020チェーンの改善）に振り向けるのが妥当。
