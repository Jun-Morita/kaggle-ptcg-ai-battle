# exp056 — 学習タイブレーカー（SEARCH_PRI型のLO適用） → NO-GO（教師に信号なし、学習前に棄却）

## 仮説
exp054-Dの棚卸しで、LOパイロット(koff)は1試合あたり約6決定でトップ同点タイ＝実質恣意選択
（TO_HAND 53%・TO_ACTIVE 59%・SWITCH 41%）。SEARCH_PRI3（v019, z=2.40）と同じ前提条件なので、
silver圏の同型デッキ使い（MR.h 965 / Takuma_Tsuji 938）のリプレイを教師に、
**タイ集合内のみで発言権を持つ**小型チューザーをBCすれば上位互換パッチになるはず。

## 実施
1. 教師リプレイ全量取得: MR.h(sub 54604969, 90試合) + Tsuji(sub 54288544, 88試合)。
   ※両者の実戦成績~0.44-0.46は「弱い教師」ではなく**収束の定義**（不動点では勝率0.5）。
2. `extract_ties.py`: 教師の各決定に対し我々のkoffパイロットのスコアを再計算（計装法は
   probe_voids と同一、action/obsペアリングは exp043 の t/t-1 修正を踏襲）、
   タイ幅≥2かつ教師の選択がタイ集合内のレコードのみ抽出。
   → **971レコード/178試合**（TO_HAND 543・TO_ACTIVE 227・SWITCH 161）。
   optionは(area,playerIndex,index)参照——カードID復元は PD._area_list 経由（初版バグ修正済み）。

## 結果: 学習開始前のベースライン診断で仮説が死んだ

| context | n | 平均タイ幅 | **現行挙動(tie[0])一致** | 最頻カード一致 |
|---|---:|---:|---:|---:|
| TO_HAND | 543 | 3.37 | **1.000** | 0.429 |
| TO_ACTIVE | 227 | 2.61 | **1.000** | 0.388 |
| SWITCH | 161 | 2.50 | **0.981** | 0.199 |
| SETUP_ACTIVE | 31 | 2.06 | **1.000** | 0.774 |

**教師の選択は、我々のタイ集合内では ~100% 「我々と同じ先頭選択肢」。**
機構: 教師も同じ公開パイロット由来のコードで、choose()の安定ソート＝先頭選択という
決定的タイ破りまで同一。タイは「恣意」ではなく「全員が同じ側に倒す決定」だった。
彼らとのdecision-match差(0.34)は**タイ集合の外**（スコア改変による順位変化）にあり、
その模倣は exp054-C で counterfactual NO-GO 済み。

## 判定: NO-GO（教師由来のタイブレーク信号は存在しない）
- 代替教師: 原作者(1083.6)は現LBに不在。より強いLO使いは1100超に存在しない（gold帯スカウト）。
- 代替信号: 自己対戦の勝敗相関（6決定/試合×2値報酬＝信号雑音比が絶望的）、価値関数
  （exp033/045で2度失敗）——いずれも既知の失敗経路のため追わない。
- **「RLは使えるか」への最終回答を強化**: 唯一の実証済みレシピ(SEARCH_PRI型)の前提
  「空白＋教師信号」のうち、LOでは空白はあるが**教師信号が原理的に存在しない**
  （場の同型デッキ使いが全員同じコード系統のため）。

## 価値
- 学習投資ゼロで棄却できた（データ診断ファースト、SEARCH_PRI3の時と同じ手順の勝利）。
- Strategyレポート素材: 「公開コード系統が場を占めるラダーでは、同型上位者は教師にならない
  （コード同祖＝差分は改変箇所のみ）」という一般化。
- 資産: extract_ties.py（任意パイロットのタイ集合抽出器）、教師リプレイ178試合キャッシュ。
