# exp058 — v025ミラー特化の重みチューニング → NO-GO（事前基準で棄却、18件目）

## 仮説
myso1987の帯域メタ: 900-999（silver圏）は**Alakazamミラー43.4%**、v025の実戦ミラー0.35。
v025パイロットは`WEIGHTS`辞書（69ノブ、決定時にW[...]参照）＋`alak_w.json`上書き機構を持つ
＝作者のmemeticチューニングの続きをミラー特化でやれば、+0.15級のレバーになるはず。

## 方法
- 候補（ロード後にWEIGHTS.update）vs stock、CRN共通シード、swap-pair共有。
- 健全性: stock vs stock = 0.483 ≈ 0.5 ✓
- スクリーン: ミラー関連9ノブ×{0化, 2倍} = 18アーム、n=60。
- 事前固定の判定基準: **n=300でwr≥0.55（z≥1.7）なければNO-GO**。

## 結果
- スクリーン(n=60): 最大0.567（boss_killx2, mine_counterx2）＝z≈1.0でノイズ圏。
  **唯一明確な信号は負方向**: `xerosic=0`→0.317（Xerosicはミラーでも生命線、既定調律は良い）。
- 確定(n=300): boss_killx2 **0.487** / mine_counterx2 **0.513** / 合成 **0.483** ——全て~0.5、
  n=60の上振れは全てノイズ（小標本偽陽性の教訓の再演、エスカレーション規律が再び捕捉）。

## 判定: NO-GO
- 単ノブ（および上位合成）の重み摂動では、memetic調律済みの既定値からミラーを動かせない。
  「調律済みパイロットへのパッチは効かない」則の**重み空間版**。
- ミラーの残りレバーはデッキレベル（Hammer増量等）だがdeck⊗pilot（パイロットの重みは
  自デッキ前提で調律済み）＋比率実験の失敗履歴により追わない。
- silver確実化は**リロール運用＋v025の自然収束**（ミラー0.5前提でも900台は抜け得る:
  ミラーは定義上双方0.5なので、非ミラー57%での優位が不動点を決める）に委ねる。
