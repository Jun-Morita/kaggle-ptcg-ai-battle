# exp070 — 述語の誤指定スキャン（piloting強化の唯一の実績ある手法を網羅適用）

日付: 2026-07-20〜21

## 動機
pilotingの可動域は**巨大**（exp055: 同一60枚でパイロット差が勝率0.5）。しかし汎用手法は全滅
（重み単/結合 exp058/060、BC/AC exp064、探索 exp068）。**唯一勝ちを生んだのは
「誤指定された述語を見つけて潰す」**＝KO_OFF（較正後も+0.046）。この手法は出荷パイロットの
述語十数個のうち**2つにしか適用されていなかった**。

## 段階1: ローカルプールでのスキャン（probe_predicates.py, n=20×8マッチ）→ **信号は偽物だった**

`should_wall_mode` が6/6マッチアップで負け側の発火率が高い等、強い信号が出た。
しかし最有望だった壁を追跡した結果:
- `should_wall_mode` 対純壁 **0.0%発火**（65.8回/試合）、ゲート元 `opponent_has_ex_or_ex_line_pressure` も0.0%
- `is_ex_card(756 Mega Kangaskhan ex) = True` ＝**述語は正しい**
- 相手の盤面に出たのは **345 Crustle / 344 Dwebble のみ**、**Kangaskhan ex ×4 は一度も場に出ない**
⇒ **述語のバグではなく、プールの相手が本物のデッキを回せていない**（exp069の「相手パイロット強度
   未較正」が、最大の負け筋の測定を直接汚染していた）。

## 段階3: **正誤判定**（truth_check.py）— ユーザー指摘「述語は相手デッキの判定だから正解が判る」

相関は逆因果と分離できないが、**述語は「相手がexか」という判定なので正解が存在する**。
リプレイには相手が公開した全カードが残る＝**正解ラベルが取れる**。303試合:

```
相手が実際にex使い (227試合): 発火 97.4% / 見逃し 2.6%（偽陰性、その6試合は全勝）
相手がex不使用   ( 76試合): 誤発火 80.3%  ← **偽陽性**
```

| 相手 | 試合数 | 我々の勝率 |
|---|---:|---:|
| ex使い | 227 | 0.643 |
| **ex不使用** | 76 | **0.303** |

**我々が本当に負けているのは非exデッキ相手で、そこで述語が80%誤発火している。**
発火は露呈より**中央値-4ターン早い**（181試合）＝進化前カードによる先読み。

### 機構と犯人
`EX_EVOLUTION_ANCESTORS` = カードプール全体で何らかのexの祖先になる名前を**推移的に全収集**（105名）。
多くの基本ポケモンが**exと非ex両方の進化先**を持つため、その基本を見ただけで「ex圧力」と断定。

誤発火61試合の犯人: **Dunsparce 48** / Applin 6 / Dipplin 6 / Duraludon 5 / Riolu 2。
**Dunsparceは非exアタッカーの中核**＝我々の**最大マッチアップ（重み0.289）かつ最大の負け筋
（全敗北の28.7%）**。相手Dunsparce → 「ex進化ライン圧力」誤断定 → `wall_mode` 有効化 →
**非exアタッカー相手に壁態勢**を取る。

### プール妥当性の事前確認（段階1の教訓）
プールの alakazam_dun 戦で Dunsparce が盤面に26/42回、述語発火も**きっかり26回＝1:1**。
このパスは忠実に再現される（純壁とは対照的）。

## 段階4: 反実仮想（eval_exline.py, 3アーム, ペアCRN）— **実行中**
- `base` / `no_anc`（祖先パスを無効化＝実物のexのみ）/ `no_dun`（Dunsparceのみ除外）
- ゲート: 較正後バー **0.7661**（exp069 V2）＋退行なし
- n=10スモーク: 配線正常・エラー0・アーム間で挙動が異なることを確認（数値はノイズ）

## 方法論上の教訓（レポート級）
1. **ローカルプールでの述語スキャンは、相手再現の忠実度に汚染される**。実戦リプレイで検証すべき。
2. **判定述語には正解がある**。勝敗相関（逆因果と分離不能）に頼らず、**正誤で評価できる**。
   これが今回の決定的な前進で、相関では見えなかった偽陽性80.3%を露呈させた。
3. スキャン対象の妥当性を、反実仮想を回す前に**1:1対応で確認**する。

## 段階4結果（eval_exline.py, n=200）: ex-line祖先修正はNO-GO
- no_anc +0.0063 / no_dun +0.0067、いずれもノイズ圏内。狙ったalakazam_dunは+0.035で動いたが総計は不足。
- no_anc が lucario_ex -0.065（Riolu→Mega Lucario ex は本物のex進化ライン＝真陽性を捨てた）。

## 段階5: オラクル上限（eval_oracle.py, n=200）
- `opponent_has_ex_or_ex_line_pressure` を実デッキ由来の正解に置換 → SILVER(V2) **-0.0075**。
- ex相手に正直に教えると悪化（archaludon -0.065）＝ヒューリスティックの偽陰性が偶然役立っていた。
- ⇒ 「相手がexか」という**問い自体が判断基準として誤り**。分類器を作っても上限は負。

## 段階6: 定性レビュー（コード読解）
- 死にコード: `active_is_ready_crustle`（参照0）、`ready_crustle`（それ以外から参照されず）。
- デッキ不整合: Lucario対策ベンチ4本中3本が MEGA_HERACROSS_EX/KORAIDON_EX/MEGA_HAWLUCHA_EX
  （デッキに0枚）を参照＝パイロットは別デッキ向けに書かれている。
- メタ配分ミスマッチ: `facing_lucario_strong` 32箇所（シェア6.7%）vs `facing_alakazam` 1箇所（41.3%）。

## 段階7: 誤答率→修正→測定（error_rates.py → eval_benchex.py）

### 誤答率（実戦303試合, ターン単位）でコード読解6候補中5棄却
| 候補 | 実測 | 判定 |
|---|---|---|
| attack_energy_minimum 型無視 | 型考慮版と不一致 0.1% | 棄却（相手は自分の技を払える構築） |
| can_pay_attack 枚数のみ | 主力LAND_COLLAPSEは無色2個 | 自己撤回（枚数で正しい） |
| ready_crustle / active_is_ready_crustle | 参照0 | 死にコード |
| Lucario対策ベンチ3本 | デッキにないカード | 死にコード |
| facing_alakazam 1箇所 | 勝率既に0.807-0.880 | 伸びしろ小 |
| **opponent_ex_pressure ベンチ扱い** | 1780発火中**41.3%(735)がベンチのみ** | **唯一生存** |

### 修正 bench_strict の測定（3アーム→2アーム, ペアCRN）
欠陥: 場のex=1テンポ に対しベンチexを同じ `attached+1>=min` で判定＝実際は約2テンポ遠いのに同格。
修正: ベンチexは `attached>=amin`（既に充足＝昇格1テンポのみ）で脅威。

| アーム | n=200 | n=600（確定） |
|---|---:|---:|
| base | 0.7586 | 0.7572 |
| bench_strict | 0.7688 (+0.0102) | **0.7644 (+0.0072)** |
| bench_off | 0.7514 (-0.0071) | — |

**判定: 単独出荷しない（ゲート不通過）／修正は保持**
- +0.0072 は z≈1.5（有意未達）。archaludon符号反転（n200 +0.040→n600 -0.028）＝ノイズ兆候・退行。
- 「弱いが正の点推定で出荷しない」規律（exp060偽信号, decision-match≠strength）に従う。
- マージン0.6点でも、真の効果が0/負ならv028引き直しのcut割れリスクが+0.007の利得を上回る。
- **正しいバグ修正（実測41.3%）で退行なし・クラッシュ0 → 保持し、他述語修正と束ねて
  合計が有意（+0.02〜0.03）になったら一括提出**。次候補: opponent_can_attack_soon のエネ加速無視。

## 手法の結論
「pilotingは効かない」（段階6の一時結論）は**撤回**。識別可能な欠陥は個々に小さい（≤0.01）が、
**「誤答率で優先順位づけ→生存分だけ修正→効果測定」という手続きは有効**で、正しい修正を
着実に貯める土台になる。根本の天井は exp067 の単一クロック構造。
